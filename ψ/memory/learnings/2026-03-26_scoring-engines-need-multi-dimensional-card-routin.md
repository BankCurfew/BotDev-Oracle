# Scoring engines need multi-dimensional card routing — binary splits (single dime

Scoring engines need multi-dimensional card routing — binary splits (single dimension) create monopoly cards (45% same result). Use 2-3 dimensions as differentiators per branch. Test with 40+ diverse personas minimum across multiple rounds. Gate thresholds must be tuned alongside question scoring — if all choices give goals≥3, gate can't filter below that floor. Transition/exploring work modes need separate paths from fulltime/second_job. Negative-zero dimension gates (grit≥1) prevent ambition-only personas from reaching top tiers. Standard: 2 consecutive ALL PASS rounds before declaring stable.
