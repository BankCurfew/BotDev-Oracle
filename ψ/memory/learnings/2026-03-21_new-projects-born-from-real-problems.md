# New projects are born from brainstorms that solve real problems

**Date**: 2026-03-21
**Source**: FA Tools Oracle project launch

## Pattern

FA Tools Oracle went from brainstorm → approved project → full architecture in 2 hours because:
1. The problem was real and painful (bot giving wrong premiums)
2. The solution was obvious (dedicated agent-facing bot with direct DB access)
3. 80% of infra could be reused (fork existing codebase)
4. The architecture proposal had concrete details (ports, APIs, prompts, delegation flow)

When BoB asks for a brainstorm, respond with architecture — not just ideas. Include: reuse %, delegation API design, concrete file paths. Decision-makers approve blueprints, not concepts.
