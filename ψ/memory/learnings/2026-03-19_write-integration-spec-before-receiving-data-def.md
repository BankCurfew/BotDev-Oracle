# Write integration spec before receiving data — define schema, format, quality ru

Write integration spec before receiving data — define schema, format, quality rules upfront. When expecting data from other teams, write a full spec BEFORE they start work: schema fields, data types, JSON format with copy-paste examples, quality rules, delivery method. Like an API contract — agree on interface before implementation. Prevents back-and-forth reformatting and bad data entering the system.
