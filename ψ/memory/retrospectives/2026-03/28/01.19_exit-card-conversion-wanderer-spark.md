## AI Diary

This was a surgically precise session — 35 minutes from request to complete delivery. The most satisfying part was finding the Dreamer bug that แบงค์ reported. The bug was subtle: in the `exploring` workMode, `STANDARD_AGENT` tier non-diamond users get `cardId = 16` (Dreamer) assigned directly inside `determineCard()`, without going through the adaptive cutoff gate that would set `exitCard`. This means the exit card ID never gets flagged as an exit — it flows through the normal cinematic result path (Epilogue → CardReveal → CharacterSheet). The fix was elegant: instead of just checking the gate-set `exitCard`, I added a fallback after `determineCard()` that catches any card with id 14-18 and redirects to ExitPage. Belt and suspenders.

The ElevenLabs key expiry was annoying but resolved quickly once แบงค์ dropped the new key in `.env`. I learned to check the project-local `.env` first, not just `~/.bashrc`. The curl-based generation was faster than the MCP tool anyway — two sequential calls, 640-660KB MP3s, verified as valid MPEG Layer III. No pronunciation preview possible from CLI, but the George voice on `eleven_multilingual_v2` has been reliable for Thai in previous sessions.

Writing the exit content for Wanderer and Spark required a different tone than Mirror/Sleeper/Dreamer. Those three are clear failures at gates. Wanderer and Spark are people with some merit — one has experience, the other has motivation — but neither is a fit right now. The Thai copy had to be warm and respectful: "ประสบการณ์ที่คุณมีไม่ได้สูญเปล่า" (your experience isn't wasted) for Wanderer, "ไฟที่คุณมีเป็นของจริง" (your fire is real) for Spark. These aren't failures — they're "not yet" cards.

## Honest Feedback

**Friction 1: ElevenLabs key management.** The key was in `.bashrc` (old, expired) and the new one in `fa-recruitment-quiz/.env` (project-local). The MCP ElevenLabs tool uses the env var from the shell, not project `.env`. This mismatch caused a failed attempt before I found the right key. There should be a single source of truth for API keys across MCP tools and shell — perhaps a `.env` loader in the shell profile.

**Friction 2: .gitignore blocking MP3 commits.** The repo has `*.mp3` in `.gitignore` (reasonable for most cases), but narration MP3s are intentional tracked assets. Had to use `git add -f`. A better pattern would be `*.mp3` ignore with `!public/audio/narration/*.mp3` exception, or use Git LFS for audio assets. Small friction but worth noting.

**Friction 3: Shell cwd keeps resetting.** Every `cd` command resets back to BotDev-Oracle (the invocation directory). Had to use `pushd/popd` or full paths for fa-recruitment-quiz work. This is a persistent annoyance when working across repos — the session started in BotDev-Oracle but all work was in fa-recruitment-quiz.

## Lessons Learned

1. **Fallback catch for exit cards after determineCard()** — Gate checks and card determination are separate code paths. A card can be assigned an exit card ID in determineCard() without going through the gate exit mechanism. Always add a safety net that catches exit card IDs regardless of how they were assigned.
2. **Check project-local .env before shell env** — API keys may be updated in project `.env` while shell profile still has the old one. Check both.
