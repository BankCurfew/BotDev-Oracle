The summary said "read to offset 550" but the actual content was gone. A smarter compaction could preserve key file excerpts or at least hash-based cache references.

**Friction 3: MCP env var isolation.** ElevenLabs MCP server launched with old API key. `export` in bash doesn't propagate to running MCP processes. Had to fall back to raw curl. Either MCP servers need hot-reload of env vars, or there should be a restart mechanism.

## Lessons Learned

- Confidence guards create threshold cascades — audit ALL thresholds when adding normalization
- `git add -f` is standard for committing audio when `*.mp3` is in .gitignore
- Direct API calls via curl bypass MCP env var isolation issues
- Cross-dimension scoring in gap-fill (e.g., +grit:1 on commitment questions) inflates secondary dimension counts
