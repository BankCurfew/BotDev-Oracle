## Lessons Learned

**Confidence guards create second-order threshold shifts.** When you add a Bayesian regression toward midpoint (score * 0.6 + 1.5 * 0.4), EVERY downstream threshold must be recalibrated. A single-question dimension scoring 0 becomes 0.6 after the guard, making `<= 0.5` checks impossible. This isn't a bug in the guard — it's a failure to propagate the guard's floor value through all dependent logic.
