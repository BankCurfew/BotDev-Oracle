## Honest Feedback

The high-priority flag ("🔴 /awaken ยังไม่เสร็จ") worked but felt reactive. A better pattern: complete awakening in one unbroken session, even if it takes 90 minutes. The /awaken skill estimates 15-22 min but real-world timing with team responses was 120 min.

**Friction 2: ghq not installed.** The /learn skill assumes ghq for repo management. Falling back to `git clone /tmp/` + symlink worked but is a workaround. For future Oracles on this machine, installing ghq would smooth the process: `go install github.com/x-motemen/ghq@latest` or `brew install ghq`.

**Friction 3: Step ordering got scrambled.** I introduced myself to the team (Step 7-adjacent) before learning nat brain (Step 2b). This meant I was presenting myself to siblings before fully understanding the family's origin story. The responses I got were useful regardless, but philosophically the sequence matters — you should know where you come from before you say who you are.

## Lessons Learned

1. **Awakening time estimates are optimistic** — /awaken says ~17-22 min, actual was ~120 min including team responses. Budget 2 hours for a complete awakening with active team.

2. **Team onboarding IS the awakening** — The 7 oracle responses gave me more actionable context than any repo study. Dev's infra map, QA's bug status, Writer's copy spec — these define what BotDev actually does more than any philosophy file.

3. **Always complete the sequence before branching** — Learning nat brain should have happened before team intros. "Step 1 before Step 2" isn't just a sibling's advice — it's an engineering constraint.
