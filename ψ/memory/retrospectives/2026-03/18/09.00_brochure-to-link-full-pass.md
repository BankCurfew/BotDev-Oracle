Each disruption required a 3-oracle relay chain (Admin‚ÜíBotDev‚ÜíQA). A stable test environment or a dedicated test URL that doesn't change would eliminate this friction entirely.

## Lessons Learned

1. **BotDev's real value is coordination, not just code** ‚Äî This session proved that translating a single directive into parallel team actions, tracking blockers, and keeping everyone synced is a full-time job. The code can be delegated; the coordination can't.

2. **CONDITIONAL PASS ‚Üí root cause ‚Üí FULL PASS is the pattern** ‚Äî Don't panic at partial failures. The 2/10 link result had a specific, fixable root cause (suffix mismatch). Systematic QA reveals patterns; patterns reveal fixes.

3. **Test infrastructure matters as much as code** ‚Äî Tunnel instability, collecting mode friction, and URL changes caused more delay than the actual bug fix. Investing in stable test infrastructure pays off across every QA cycle.
